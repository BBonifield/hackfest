QuickLeft Hackfest :: April 2011
JavaScript

Help a zombie whale deliver a message while drinking beer and playing with JavaScript.

This month's hackfest is a bit different than any before. The theme of this month's hackfest is JavaScript. We've structured this little task to help everyone jump into some code. The task is simple, so even if you have never written JavaScript, you will have a chance to try it out and maybe learn some stuff! And for the JavaScript masters out there, this is your turn to really show off some craziness.

All project files are included. In this project, you can add your markup/css in "project.html", css in "resources/project.css" and write your JavaScript in "thepower.js". jQuery is included, but if you are feeling really 1337, try this without a framework and see how much you can optimize your filesize.

Warning: This task is ridiculously silly. If you are lame, or don't like zombie whales - don't try it. You will only hurt yourself! If you still want to participate in some JavaScript hacking, but aren't interested in playing with whales, talk to Sam about contributing to open source JavaScript projects, like jQuery!

Zombie whale lives under the sea, and his job is to deliver messages. Since there is no internet in the ocean, everyone relies on zombie whale to deliver their messages.

Zombie whale only knows one language: JavaScript. He needs your help to write instructions for his delivery.

Part I

First, zombie whale must go to hipster octopus to collect his message. Your JavaScript instructions should show zombie whale (#zombie_whale) how to swim over to hipster octopus (#hipster_octopus) and ask him to fill out his message (#message_form). Don't forget to write hipster octopus's message to zombie whale's memory!

Wait, what?

The first step to showing zombie whale how to swim is knowing how to make him move with JavaScript. There are many JavaScript libraries available to help you move him around. jQuery is already built into the project skeleton. Zombie whale is contained in a div with an ID of 'zombie_whale'. Check out jQuery Animate. Be sure to add a callback so zombie whale knows to do something once he gets there. ( check this out for help using a callback with jQuery animate() )

#message_form is hidden. Try using some css to change it's visibility.

Once the form is filled out, remember to save the values into memory

Wanna be pro?

Too simple? Why don't you pair your JavaScript handlers with some CSS3 animations to ease up on zombie whale's processor. Here's some inspiration.

Hmmm, what else? Maybe you should take a closer look at #zombie_whale? What's that there? Is that a sprite? Zombie whale just got a whole lot classier.

Wanna get real fancy? How about adding controls and letting us humans drive zombie whale around? Ooooh yeah!

Part II

Now that zombie whale has hipster octopus's message, he must swim and deliver it to the recipient (#big_squid, #star_starfish, or #drunk_clam).

Huh? How do I do that?

Hopefully you saved the recipient name in Part I. Now we need to find the right sea creature to deliver it to. First, it would be helpful to check out the position of zombie whale's target.

Once zombie whale finds his recipient, he needs to deliver the message! Maybe using jQuery text() will help zombie whale convey his message.

It's only 7 o'clock and this is boring.

Ok then, "JavaScript Fancy Pants", why don't you change zombie whale's life and teach him how to make clones of himself? Create an Asynchronous Zombie Whale Delivery System. Hypherthreading zombie whales FTW!

Still want more to do? How about making the sea creature recipients more realistic and teach them how to swim around the ocean.

Okay, more ideas? Try making the ocean a canvas and animate some stuff. Or try making a game out of it. Maybe this could be applied to a real-life challenge. Anything rediculous will do.
